항목 요구사항

중요

정기적인 재시작 없이는 성능 저하가 발생합니다.

개인 PC에서 서버 호스팅

설정 단계

  1. Java 17 64-bit 설치 - Adoptium 또는 Oracle에서 다운로드
  2. 서버 파일 추출 - CurseForge에서 서버 파일 다운로드 후 원하는 폴더에 추출
  3. 서버 실행
    • Windows: run.bat 실행
    • Linux/Mac: run.sh 실행
  4. EULA 동의 - eula.txt 파일을 열고 eula=false를 eula=true로 변경
  5. 명령 블록 활성화 - server.properties에서 enable-command-block=true 설정
  6. 포트 포워딩 - 라우터에서 25565 포트 포워딩 설정 (외부 접속용)

LAN 플레이 (Essential 모드)

Essential 모드를 사용하면 서버 없이 친구와 함께 플레이할 수 있습니다.

  1. Essential 모드가 이미 포함되어 있음
  2. 게임 내에서 친구 추가
  3. 월드 열기 → 친구 초대

서버 업데이트

중요: 백업 먼저!

world 폴더와 server.properties를 반드시 백업하세요.

업데이트 절차

  1. 기존 폴더 삭제: config, defaultconfig, kubejs, mods, scripts
  2. 새 모드팩 버전에서 위 폴더들을 복사
  3. 백업해둔 server.properties 복원
  4. 서버 재시작

관리자 명령어

명령어 설명
/flan bypass 클레임(점유) 영역 강제 진입
/spark profiler open 렉 원인 분석 도구 실행
/cu inventory read <플레이어> 플레이어 인벤토리 확인
/pregen start <반경> 월드 청크 미리 생성

성능 최적화 팁

  • 청크 미리 생성: /pregen 명령어로 플레이 영역 청크를 미리 생성
  • 정기 재시작: 6시간마다 자동 재시작 스크립트 설정
  • 플레이어 제한: 서버 사양에 맞게 최대 플레이어 수 제한
  • 뷰 거리 조정: server.properties에서 view-distance 조정 (8-10 권장)
